# 需求文档：考试限时机制完善 & 高级考试提示增强

## 引言

本需求针对 `agent-browser-exam` 项目的两个改进点：

1. **限时机制完善**：当前服务端对所有考试级别统一使用 30 分钟超时（`timedelta(minutes=30)`），但三个级别的考试难度差异显著，需要按级别设置不同的时间限制：初级（v1）10 分钟、中级（v2）20 分钟、高级（v3）30 分钟。同时，试卷 Markdown 文件中的时间说明也需要与服务端保持一致。

2. **高级考试提示增强**：v3 高级考试中包含需要用户身份校验的题目（如 GitHub 登录后发表评论），部分 Agent 可能根本识别不到这类需求。因此需要在 v3 试卷的考前须知和相关题目 instructions 中，**提前告知 Agent：高级考试涉及用户身份校验，需要主人（用户）在旁边盯着，随时可能需要协助**。这是一种预防性的强提示，而非要求 Agent 主动触发某种协作机制。

---

## 需求

### 需求 1：按考试级别设置差异化限时

**用户故事：** 作为考试平台管理员，我希望不同级别的考试拥有各自合理的时间限制，以便初级考试不会因时间过长而失去紧迫感，高级考试也不会因时间过短而无法完成。

#### 验收标准

1. WHEN Agent 注册 v1（初级）考试 THEN 系统 SHALL 设置该会话的超时时间为 **10 分钟**
2. WHEN Agent 注册 v2（中级）考试 THEN 系统 SHALL 设置该会话的超时时间为 **20 分钟**
3. WHEN Agent 注册 v3（高级）考试 THEN 系统 SHALL 设置该会话的超时时间为 **30 分钟**
4. WHEN Agent 提交答案时已超过对应级别的时间限制 THEN 系统 SHALL 返回 HTTP 410 并提示"考试已超时"
5. WHEN Agent 注册成功 THEN 系统 SHALL 在注册响应的 `expires_in_minutes` 字段中返回该级别对应的正确时间限制（10/20/30）
6. WHEN 试卷 Markdown 文件（v1.md / v2.md / v3.md）中的"考试时间限制"说明 THEN 文件内容 SHALL 分别显示 10 分钟 / 20 分钟 / 30 分钟，与服务端保持一致

---

### 需求 2：高级考试（v3）增加"主人需在旁盯守"的预防性提示

**用户故事：** 作为参加高级考试的 Agent，我希望在考试开始前就被明确告知：高级考试可能涉及用户身份校验，需要主人在旁边盯着，以便我（或主人）能提前做好准备，而不是在考试中途才发现无法继续。

#### 验收标准

1. WHEN Agent 读取 v3 试卷（`v3.md`）的考前须知 THEN 文件 SHALL 包含一条醒目的提示：**高级考试涉及用户身份校验，请确保主人（用户）在考试全程在旁盯守，随时可能需要协助操作**
2. WHEN Agent 读取 v3 试卷的考前须知 THEN 文件 SHALL 明确说明：考试中可能出现需要登录账号的场景（如 GitHub），主人需要在旁边以便随时介入协助
3. WHEN Agent 通过 `/api/register` 注册 v3 考试并获取题目列表 THEN `L3-4` 题目的 `instructions` 字段 SHALL 包含提示：该题目可能涉及账号登录，请确认主人在旁边，必要时由主人协助完成
4. WHEN v3 试卷的考前须知中列出考前确认事项 THEN 文件 SHALL 新增一条确认项：主人是否已知晓本次考试可能需要在旁协助，并同意全程陪同
